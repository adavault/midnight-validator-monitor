# Release Notes - v1.0.5

**Release Date:** 2026-02-21
**Type:** Patch release (reliability)

---

## Summary

Daemon mode now retries the initial RPC connection with exponential backoff instead of crashing immediately when the node isn't available. This prevents systemd restart storms after server reboots.

---

## Problem

After a server reboot on vdumdn57, `mvm-sync --daemon` crashed and was restarted by systemd **466 times** over 79 minutes because the Midnight node took longer to start than the mvm service. Each crash produced ~6.5 error lines, totaling 3,026 lines of noise in the journal. The daemon was exiting on the very first RPC failure at startup, relying entirely on systemd `Restart=on-failure` for recovery.

## Fix

When running in daemon mode (`--daemon`), the sync command now retries the initial `get_chain_tip` RPC call with exponential backoff:

- **Backoff schedule:** 10s → 20s → 40s → 60s (capped)
- **Signal-aware sleep:** Respects SIGTERM/SIGINT/SIGQUIT during backoff waits for clean shutdown
- **Logging:** WARN level messages show the retry delay and cause, INFO on successful connection

Non-daemon mode is unchanged — it still exits immediately with a helpful error tip, which is the correct behavior for interactive use.

### Example journal output (daemon waiting for node)

```
WARN Node not available at http://localhost:9944 (connection refused), retrying in 10s...
WARN Node not available at http://localhost:9944 (connection refused), retrying in 20s...
WARN Node not available at http://localhost:9944 (connection refused), retrying in 40s...
INFO Connected to node at http://localhost:9944
INFO Starting block synchronization
```

## Upgrade Instructions

### From GitHub Releases

```bash
# Download and install (service will auto-restart)
curl -LO https://github.com/adavault/midnight-validator-monitor/releases/download/v1.0.5/mvm-x86_64-unknown-linux-gnu.tar.gz
tar xzf mvm-x86_64-unknown-linux-gnu.tar.gz
chmod +x mvm
sudo ./mvm install
```

### Verify

```bash
mvm --version
# Should show: mvm 1.0.5

systemctl status mvm-sync
# Should show: active (running)
```
